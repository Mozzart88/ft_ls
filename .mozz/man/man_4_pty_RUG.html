PTY (4) Руководство по драйверам устройств PTY (4)

ИМЯ

pty -
драйвер псевдо-терминала

СИНТАКСИС

pseudo-device pty[ количество ]

ОПИСАНИЕ

ptyДрайвер обеспечивает поддержку для устройства пары называют псевдо терминал . Псевдотерминал - это пара символьных устройств, ведущее устройство и подчиненное устройство. Подчиненное устройство предоставляет процессу интерфейс, идентичный описанному в tty (4) . Однако, в то время как все другие устройства, которые предоставляют интерфейс, описанный в tty (4), имеют какое-то аппаратное устройство позади них, ведомое устройство вместо этого имеет другой процесс, управляющий им через главную половину псевдотерминала. То есть все, что записано на ведущем устройстве, передается на ведомое устройство в качестве ввода, а все, что записывается на ведомом устройстве, представляется как ввод на ведущем устройстве.

При конфигурировании, если в спецификации указан необязательный счетчик, конфигурируется это количество пар псевдо-терминала; количество по умолчанию составляет 32.

Следующие вызовы ioctl (2) применяются только к псевдо-терминалам:

TIOCSTOP
Останавливает вывод на терминал (например, набрав ' ^S'). Не принимает никаких параметров.
TIOCSTART
Перезапускает вывод (останавливается с помощью TIOCSTOPили набирая ' ^S'). Не принимает никаких параметров.
TIOCPKT
Включить / отключить пакетный режим. Пакетный режим включается путем указания (посредством ссылки) ненулевого параметра и отключается путем указания (посредством ссылки) нулевого параметра. При применении к ведущей стороне псевдотерминала каждое последующее считывание (2) с терминала будет возвращать данные, записанные на подчиненной части псевдотерминала, которому предшествует нулевой байт (обозначенный символом как TIOCPKT_DATA), или один байт, отражающий состояние управления. Информация. В последнем случае байт представляет собой включающий или ноль или более битов:
TIOCPKT_FLUSHREAD
всякий раз, когда очередь чтения для терминала сбрасывается.
TIOCPKT_FLUSHWRITE
всякий раз, когда очередь записи для терминала очищается.
TIOCPKT_STOP
всякий раз, когда вывод на терминал останавливается а-ля ' ^S'.
TIOCPKT_START
всякий раз, когда вывод на терминал перезапускается.
TIOCPKT_DOSTOP
всякий раз, когда t_stopc равен ' ^S', а t_startc равен ' ^Q'.
TIOCPKT_NOSTOP
всякий раз, когда начальный и конечный символы не равны ' ^S/^Q'.

В то время как этот режим используется, наличие информации о состоянии управления, которая должна быть прочитана со стороны ведущего, может быть обнаружено посредством выбора (2) для исключительных условий.

Этот режим используется rlogin (1) и rlogind (8) для реализации удаленного эхолота, локального ^S/^Q«управляемого потоком» удаленного входа в систему с надлежащей обратной очисткой вывода; это может использоваться другими подобными программами.

TIOCUCNTL
Включите / отключите режим, который позволяет небольшому количеству простых пользовательских команд ioctl (2) проходить через псевдотерминал, используя протокол, аналогичный протоколу TIOCPKT. TIOCUCNTLИ TIOCPKTрежимы являются взаимоисключающими. Этот режим включается со стороны мастера псевдотерминала путем указания (посредством ссылки) ненулевого параметра и отключается путем указания (посредством ссылки) нулевого параметра. Каждое последующее чтение (2) со стороны ведущего будет возвращать данные, записанные на подчиненной части псевдотерминала, которому предшествует нулевой байт, или один байт, отражающий операцию управления пользователем на ведомой стороне. Пользовательская команда управления состоит из специального ioctl (2)работа без данных; команда задается как UIOCCMD(n), где n - это число в диапазоне 1-255. Значение операции n будет получено в виде одного байта при следующем чтении (2) со стороны ведущего. IOCTL (2) UIOCCMD (0) является не-оп , который может быть использован для зондирования для существования этого объекта. Как и в TIOCPKTрежиме, командные операции могут быть обнаружены с помощью выбора (2) для исключительных условий.
TIOCREMOTE
Режим для основной половины псевдотерминала, независимо от TIOCPKT. В этом режиме вход псевдотерминала управляется потоком и не редактируется (независимо от режима терминала). Каждая запись в управляющий терминал создает границу записи для процесса, считывающего терминал. При обычном использовании запись данных похожа на данные, введенные в виде строки на терминале; запись в 0 байтов похожа на ввод символа конца файла. TIOCREMOTEможет использоваться при удаленном редактировании строки в диспетчере окон или когда требуется ввод с управлением потоком.
